### 1 — `factor_order = "mixed"` default

**What changed in the package:**

The behavior of `factor_order` in `ternD()` and `ternG()` has been updated. There is now an option called `"mixed"`: it orders two-level categorical variables (e.g. Male/Female) alphabetically — or by factor level order if the variable is a factor — so the reference group is always displayed first. For variables with three or more levels, it orders by decreasing frequency unless the variable is a factor with explicitly defined levels, in which case that level order is respected.

**App changes needed:**

`"mixed"` for factor ordering for `ternD()` and `ternG()` — make this the APP default (this is also the FUNCTION default so it can actually just silently pass if you want it to). This is universally applied whether `ternD` is used, or `ternG` for a two group table, or `ternG` for a three+ group table

### 2 — OR for two-level categoricals + consolidated Factor Ordering step

**What changed in the package:**

Another new feature has also been added which must be understood: Previously, odds ratios were computed only for variables TernTables auto-detected as binary — specifically columns whose values were Y/N, YES/NO, or numeric 0/1. Two-level categorical variables like Male/Female or Present/Absent were displayed with the standard hierarchical sub-row format but received a dash (-) in the OR column, because the binary detection logic did not recognize them as eligible for OR computation....... Odds ratios are now computed for any two-level categorical variable regardless of the labels used. When OR_col = TRUE, the reference group (determined by factor level order, or alphabetical order for non-factors) displays 1.00 (ref.) in the OR column, and the non-reference row displays the computed OR with 95% CI in the standard format (e.g. 1.34 [0.98–1.83]). Fisher's exact or Wald method is chosen automatically using the same Cochran expected-cell criterion applied to all other categorical variables. Variables with three or more levels continue to display - in the OR column, as OR is not defined for multinomial comparisons without a specific contrast.

### 3 — Post-hoc pairwise comparisons: `post_hoc` parameter

**What changed in the package:**

A new parameter `post_hoc` has been added to ternG(). When post_hoc = TRUE, pairwise post-hoc comparisons are automatically performed for three-group (or more) comparisons on continuous and ordinal variables, but only when the omnibus p-value is significant (p < 0.05). The test used for post-hoc is matched to the omnibus test that was run: if the variable went through the parametric path (Welch ANOVA), Games-Howell is used for pairwise comparisons; if it went through the non-parametric path (Kruskal-Wallis), Dunn's test with Holm correction is used. Categorical variables never receive post-hoc testing regardless of this setting. Results are displayed as compact letter display (CLD) notation — superscript letters appended directly to each cell value (e.g. "72.4 ± 8.1ᵃ") — where groups sharing a superscript letter are not significantly different from each other after correction. The CLD is computed via the multcompView package (Piepho 2004 connected-letters method). A footnote is automatically added to the Word output explaining the superscript convention, naming the specific variables that received post-hoc testing, and identifying the test used — but only when at least one post-hoc test actually ran. If the user also supplies their own table_footnote, the CLD footnote is placed first and the user's footnote follows. post_hoc defaults to FALSE and requires both rstatix and multcompView to be installed. It has no effect on two-group tables.
